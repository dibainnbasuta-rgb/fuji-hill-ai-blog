# Claude Handoff

最終更新: 2026-06-17

このサイトは「AIと喧嘩しながら富士ヒルゴールドを目指すブログ」です。
CodexとClaudeの両方で更新する可能性があるため、作業前後にこの内容を確認してください。

## 公開ルール

- GitHubの`main`へpushするとCloudflare Pagesで公開されます。
- pushはユーザーが「公開してOK」と明示した時だけ行います。
- 下書きや確認前の記事はpushしません。
- ClaudeとCodexで同じファイルを同時に編集しないでください。
- 作業前に可能なら`git fetch`と`git status`で最新状態を確認してください。
- commitする時は、今回の作業に必要なファイルだけを含めてください。

## 記事追加時に更新する場所

新しい記事を追加したら、記事ページだけで終わらせず、以下も確認します。

- `articles/YYYY-MM-DD-slug.html`
- `assets/images/articles/YYYY-MM-DD-slug/`
- `index.html`
- `blog.html`
- `experiments.html`
- `sitemap.xml`
- `feed.xml`（公開push後に `python tools/blog.py feed` で再生成→追加push）
- 目次/Xシェア/OGPは `python tools/blog.py toc / share / ogp` で自動挿入
- サイドバーは公開push後に `python tools/blog.py sidebar` で再生成→追加push

TOPの最新記事は最大5件までです。
6件目以降はTOPから外し、`blog.html`やカテゴリページへ回します。

## 記事一覧の表示ルール

- 今すぐ: プルダウンは入れない。TOPは最新5件。記事一覧は全記事表示。
- 20記事まで: 記事一覧は全記事表示。
- 21〜30記事: 記事一覧は全記事表示のまま、カテゴリ導線を強める。
- 31記事以上: 記事一覧は最新20件を基本表示。必要なら「20件 / 50件 / 100件」プルダウンを追加。
- 50記事以上: 年月別アーカイブ or ページ分割を検討。

表示件数プルダウンは実装可能ですが、今は記事数が少ないため入れません。
将来入れる場合も、初期表示は軽めにしてください。

## 6つの実験カテゴリ

記事は最低1つ、自然なら複数カテゴリに入れます。
AI相談ログは独立カテゴリではなく、記事内の「AI相談メモ」として扱います。

- 外ライド・実走
- ローラー練習
- 食事と補給
- 機材とアイテム
- 旧ブログ発掘
- 体重とコンディション

## 既存記事一覧の正（source of truth）

既存記事がどのカテゴリに属するかの固定一覧は、ここには持ちません。記事を追加するたびに古くなり、二重管理になるためです（Codex 提案・2026-06-17 反映）。**既存記事一覧の正は次の3つ**とします。

- `blog.html`（全記事・新着順）
- `experiments.html`（カテゴリ別の掲載）
- Git ログ（`git log` / コミット履歴）

突き合わせ確認には `python tools/blog.py index-check`（`blog.html` と `articles/` の整合チェック）が使えます。
このファイルでは「新記事をどのカテゴリに入れるか」の運用ルール（上の6カテゴリ・最低1つ・自然なら複数・AI相談は独立カテゴリにしない）だけを管理します。

## 記事の文体

- 本文の一人称は「私」。
- 記事本文では「中の人」は基本使いません。
- 読み物として自然な実験日記にします。
- データと体感は分かりやすく残します。
- 脳内ツッコミは薄めで、くどくしません。
- 無関係な脱線は入れません。
- 上から目線の「読者向けの学び」調は避けます。
- 改行を増やしすぎないでください。基本は2〜4文で1段落くらい。
- 読点「、」も多くしすぎないでください。

## 記事構成

できるだけ以下の流れにします。

1. 日付・カテゴリ
2. タイトル
3. 短いリード文
4. メイン画像
5. 目次（`blog.py toc` の自動生成）
6. 本文
7. 必要なデータ
8. AI相談メモ（必要な場合のみ）
9. 自分用メモ
10. Xシェアボタン（`blog.py share` の自動生成）
11. 関連記事（手動キュレーション3件）

**冒頭の「まとめ」ボックスは新規記事では作りません**（ユーザー指示・2026-07-02〜）。
目次と役割が重複するためです。既存記事のまとめは削除せず、そのまま残します。

記事ページにはJSON-LDを入れてください。
ただし、見えていない情報や嘘の情報は入れないでください。
